## 2. Time is Not a Label: Continuous Phase Rotation for Temporal Knowledge Graphs and Agentic Memory

**Topics:** Agent Skill, Memory, RL, Evolve
**Authors:** Weixian Waylon Li, Jiaxin Zhang, Xianan Jim Yang, Tiejun Ma, Yiwen Guo
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2604.11544v1) | [PDF](http://arxiv.org/pdf/2604.11544v1.pdf)

### 1. 一句话结论

RoMem通过连续相位旋转和语义速度门，将时间建模为复向量空间中的几何操作，实现无需删除的时序知识图谱冲突解决和代理记忆优化。

### 2. 为什么重要

现有代理记忆系统在处理时间冲突时面临静态-动态困境：无法区分永久性事实（如出生地）和演化性事实（如总统任期），导致历史知识被遗忘或需要昂贵的LLM调用。RoMem通过几何阴影机制，使过时事实在复向量空间中相位偏移，从而自然地被当前事实超越，无需删除或排序，实现了高效的时序推理和记忆保持。

### 3. 研究问题

1. 功能时间建模是否能维持或提升标准TKGE基准上的性能？
2. RoMem能否在时间推理任务上超越现有代理记忆基线，同时保持非时间检索的鲁棒性？
3. RoMem能否泛化到未见过的领域特定关系？

### 4. 背景与问题设定

结构化记忆表示（如知识图谱）是自主代理和长期系统的核心。然而，现有方法将时间建模为离散元数据（如时间戳），导致三种困境：(i) 破坏性覆盖，永久丢失历史上下文；(ii) LLM仲裁，每次摄入都需要语言模型调用，成本高昂；(iii) 按最近性排序，会无意中埋藏旧但永久的事实。这被称为静态-动态困境，即离散元数据无法区分永久关系和演化关系，无法在不牺牲静态知识的情况下解决时间冲突。

### 5. 方法概览

- 任务 / 环境: RoMem是一个可插入的时间知识图谱模块，适用于结构化记忆系统，特别是代理记忆。它处理文本事件流，提取关系事实，并在知识图谱中维护一个追加只读的内存状态。
- 变量 / 干预设计: RoMem通过两个机制解决时间冲突：(1) 连续几何阴影：将时间建模为复向量空间中的连续几何旋转，使动态事实随时间流逝而相位偏移，而静态事实保持相位对齐；(2) 语义速度门：一个预训练的MLP，根据关系的文本嵌入预测其波动性（αr），控制旋转速度，使静态关系（αr≈0）稳定，动态关系（αr≈1）快速旋转。
- 评价指标: ['MRR (Mean Reciprocal Rank)', 'Hits@k', 'Recall@k', 'LLM-as-judge accuracy (Acc@k)']
- 模型比较: 在时间知识图谱补全任务上，RoMem-ChronoR在ICEWS05-15上达到72.6 MRR，优于所有基线模型。在代理记忆任务上，它在MultiTQ上实现了2-3倍的MRR和答案准确率提升，在LoCoMo混合基准上占主导地位，在DMR-MSC上保持静态记忆零退化，并在FinTMMBench上实现了零样本到未见过的金融领域的泛化。
- 我的理解: RoMem的核心创新在于将时间从一个离散的标签转变为一个连续的几何操作。它通过语义速度门学习每个关系的内在时间动态性，并利用连续相位旋转在复向量空间中实现“几何阴影”——过时的事实被旋转到相位上，从而在查询时自然地被当前事实超越。这种方法避免了删除、排序或昂贵的LLM调用，提供了一种优雅且高效的解决方案。

### 6. 关键发现

- 状态-动态困境的正式化: 论文正式定义了图基代理记忆中的静态-动态困境，指出离散时间元数据将所有关系同等对待，无法在不牺牲静态知识的情况下解决时间冲突。
- 连续几何阴影: 将时间冲突解决形式化为复向量空间中的连续几何阴影，用几何接近性取代破坏性的数据库更新和每次摄入的LLM调用。

## 3. Self-Evolving LLM Memory Extraction Across Heterogeneous Tasks

**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2604.11610v1) | [PDF](http://arxiv.org/pdf/2604.11610v1.pdf) | [Code](https://github.com/ayyyq/heterogeneous-memory-extraction)

### 1. 一句话结论

提出CluE框架，通过聚类分组训练样本并独立分析各簇，实现跨异构任务的稳定自演化记忆提取，显著优于现有方法

### 2. 为什么重要

随着LLM助手变得持久化和个性化，记忆提取成为关键挑战。现有静态提示和自演化框架在异构任务上表现不佳，CluE通过聚类策略解决了任务多样性带来的优化困境，提升了记忆提取的通用性和稳定性

### 3. 研究问题

1. 如何在异构任务中有效提取和保留有用信息？
2. 现有自演化框架在异构任务分布下为何表现不佳？
3. 如何设计一个能稳定学习并跨任务泛化的记忆提取策略？

### 4. 背景与问题设定

LLM助手需要从对话中提取有用信息作为记忆，但不同任务（如个性化、问题解决、代理任务）所需记忆类型差异大。现有静态提示无法跨任务通用，而现有自演化框架假设任务分布同质化，导致在异构任务下性能下降。

### 5. 方法概览

- 任务 / 环境: 在BEHEMOTH基准上评估记忆提取，该基准整合了18个数据集，涵盖个性化、问题解决和代理任务三类，使用下游效用驱动指标评估提取质量
- 变量 / 干预设计: 提出CluE框架，将训练样本按提取场景聚类，独立分析每个簇，然后合成跨簇洞察来更新提取提示
- 评价指标: ['宏观准确率 (Macro Accuracy)', '相对增益 (Relative Gain)']
- 模型比较: ['静态提示：No Memory, Simple, Mem0, ReasoningBank, OpenMemory, Survey', '自演化框架：GEPA, ACE, MemEvolve, CluE']
- 我的理解: CluE通过聚类将异构任务分组，避免了不同任务间的反馈冲突，实现了更稳定的自演化。其核心创新在于将全局优化分解为局部分析和跨簇合成，有效解决了异构分布下的优化困境。

### 6. 关键发现

- 无单一静态提示主导所有任务类别: 实验表明，没有一个静态提取提示能在所有任务类别上表现最佳，不同提示在不同类别上各有优势，凸显了异构性挑战。
- 现有自演化框架在异构任务下性能下降: GEPA、ACE、MemEvolve等框架在异构任务分布下出现性能退化，表现为在某些类别上提升的同时在其他类别上下降，无法实现跨任务的均衡改进。
- CluE显著提升跨任务泛化能力: CluE在BEHEMOTH上实现了9.04%的相对增益，且在所有三个任务类别上均取得一致提升，显著优于所有基线方法。

## 5. Relax: An Asynchronous Reinforcement Learning Engine for Omni-Modal Post-Training at Scale

**Topics:** Agent, Skill, Memory, RL, Evolve
**Authors:** Liujie Zhang, Benzhe Ning, Rui Yang, Xiaoyan Yu, Jiaxing Li, Lumeng Wu, Jia Liu, Minghao Li, Weihang Chen, Weiqi Hu, Lei Zhang
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2604.11554v1) | [PDF](http://arxiv.org/pdf/2604.11554v1.pdf) | [Code](https://github.com/rednote-ai/Relax)

### 1. 一句话结论

Relax是一个异步强化学习引擎，通过服务化架构、异步数据总线和原生多模态支持，解决了大规模多模态代理训练中的数据流异构性、可扩展性和延迟-吞吐权衡问题。

### 2. 为什么重要

随着大模型向多模态和代理工作流发展，传统强化学习训练系统面临数据异构、操作鲁棒性和延迟-吞吐权衡三大挑战。Relax通过三重协同设计，实现了高效、稳定、可扩展的多模态RL训练，为下一代智能体系统提供了关键基础设施。

### 3. 研究问题

1. 如何在多模态和代理工作流下实现高效、鲁棒的强化学习训练？
2. 如何解决异构数据流、操作鲁棒性和延迟-吞吐权衡之间的相互依赖问题？
3. 如何在不牺牲收敛性的前提下，实现训练系统的异步化以提升吞吐量？

### 4. 背景与问题设定

强化学习（RL）后训练已被证明能有效解锁大语言模型的推理、自我反思和工具使用能力。随着模型扩展到多模态输入和代理多轮工作流，RL训练系统面临三个相互依赖的挑战：异构数据流、大规模下的操作鲁棒性以及延迟-吞吐权衡。现有系统通常在文本为中心的管道上进行改造，难以有效处理图像、视频和音频等多模态数据，且在大规模运行时缺乏故障隔离和弹性扩展能力。

### 5. 方法概览

- 任务 / 环境: Relax是一个用于大规模多模态强化学习后训练的开源引擎，支持Qwen3系列模型，处理图像、文本、音频和视频等多种模态。
- 变量 / 干预设计: Relax通过三个协同设计的架构层解决挑战：1）原生多模态架构，将多模态支持构建到整个堆栈中；2）角色隔离的服务架构，每个RL角色作为独立、故障隔离的服务运行；3）服务级解耦，通过TransferQueue数据总线实现异步训练，单一延迟参数平滑地在在线策略、近在线策略和完全异步执行之间插值。
- 评价指标: 端到端速度提升（x1.20, x1.76, x2.00）、收敛到相同奖励水平、R3（Rollout Routing Replay）在MoE模型上的开销（1.9% vs. 32%）、多模态RL收敛稳定性（超过2000步视频无退化）。
- 模型比较: Relax在Qwen3-4B上比veRL快1.20倍，在完全异步模式下比colocate快1.76倍；在Qwen3-Omni-30B上快2.00倍。R3支持的开销仅为1.9%，而veRL在相同配置下有32%的性能下降。
- 我的理解: Relax的核心创新在于将RL训练系统从一个紧密耦合的单体架构转变为一个解耦的、服务化的、异步的系统。通过将每个RL角色（如Actor、Critic）部署为独立的Ray服务，实现了故障隔离和独立扩展。通过引入TransferQueue数据总线，将数据交换与执行解耦，从而允许训练和推理在不同集群上并行运行，极大地提升了吞吐量。单一的延迟参数统一了从在线到离线的训练模式，提供了极大的灵活性。原生多模态支持则确保了整个管道能够高效处理各种模态数据。

### 6. 关键发现

- 异步训练显著提升吞吐量: Relax的完全异步模式在Qwen3-4B上比colocate快1.76倍，在Qwen3-Omni-30B上快2.00倍，证明了服务解耦和异步数据总线的有效性。
- 统一的延迟-吞吐权衡控制: 通过TransferQueue的单一延迟参数，Relax能够平滑地在在线、近在线和完全异步模式之间切换，为用户提供了一个灵活的配置选项来平衡训练效率和收敛性。
- 原生多模态支持与稳定收敛: Relax在Qwen3-Omni上实现了跨图像、文本、音频和视频的稳定多模态RL收敛，能够持续处理超过2000步的视频而无性能退化。
- 高效的MoE模型支持: Relax支持R3（Rollout Routing Replay）用于MoE模型，仅带来1.9%的开销，远优于veRL的32%性能下降。
