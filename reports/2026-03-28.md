## 1. Natural-Language Agent Harnesses

**Topics:** Agent, Skill, Memory, RL, Evolve
**Authors:** Linyue Pan, Lexiao Zou, Shuo Guo, Jingchen Ni, Hai-Tao Zheng
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2603.25723v1) | [PDF](http://arxiv.org/pdf/2603.25723v1.pdf)

### 1. 一句话结论

提出自然语言代理 harness（NLAH）和智能 harness 运行时（IHR），将代理控制逻辑外部化为可执行的自然语言 artifact，实现可移植、可比较、可分析的 harness 设计。

### 2. 为什么重要

代理性能越来越依赖于 harness 工程，但现有 harness 设计通常隐藏在控制器代码和运行时特定约定中，难以转移、比较和研究。NLAH 和 IHR 将高阶控制逻辑外部化为可编辑的自然语言 artifact，使 harness 成为可科学研究的对象，推动代理系统的可解释性、可移植性和模块化设计。

### 3. 研究问题

1. RQ1: 在固定预算下，共享运行时章程和特定基准的 harness 逻辑如何改变代理行为和任务结果？
2. RQ2: 一旦模式显式化，模块是否可以在模式层面进行组合和消融？
3. RQ3: 在共享运行时下，原生代码 harness 和重构的自然语言 harness 之间仍存在哪些差异？

### 4. 背景与问题设定

现代代理的成功与否越来越依赖于其周围的 harness：一个结构化多步推理、工具使用、记忆、委托和停止的控制堆栈。尽管外部化控制模式（如 reason-act 循环、检索增强生成）已被证明至关重要，但 harness 逻辑通常分散在控制器代码、隐藏的框架默认值、工具适配器、验证器脚本和运行时特定假设中，难以转移、公平比较和干净消融。这使得评估陷入控制器捆绑比较，而非模块级证据。

### 5. 方法概览

- 任务 / 环境: 在编码（SWE-bench Verified）和计算机使用（OSWorld）基准上进行受控评估。
- 变量 / 干预设计: 通过消融实验（移除运行时技能或 harness 技能）和模块组合/消融（添加文件后端状态、证据支持的答案、验证器、自我进化、多候选搜索、动态编排模块）来评估 harness 模式逻辑的影响。

- 我的理解: 该工作通过将 harness 逻辑从代码中解耦并用自然语言表示，实现了控制逻辑的可移植性和可分析性。IHR 运行时通过解释 NLAH 并分离运行时章程与 harness 逻辑，使得在不同任务间共享和比较 harness 成为可能。实验表明，虽然添加更多结构化模块不一定能提升整体性能，但它们能显著改变系统行为，尤其是在边界案例上，这为理解代理行为提供了新的视角。

### 6. 关键发现

- RQ1: 行为效应: 共享运行时章程和特定基准的 harness 逻辑显著改变了代理行为，但对任务解决率的影响较小。Full IHR 产生的过程指标（令牌、调用、运行时间）远高于消融设置，表明其行为是真实的控制，而非简单的提示装饰。
- RQ2: 模块组合与消融: 模块效果集中在小部分边界案例上，而非均匀提升整个基准。模块可分为两类：一类（如自我进化）改进了求解循环本身；另一类（如文件后端状态）主要改善了过程结构。更明确的结构并不总是意味着更好的最终任务性能。
- RQ3: 代码到文本的迁移: 在共享运行时下，原生代码 harness 和重构的自然语言 harness 之间存在差异，但性能差异较小，表明 NLAH 能够有效地迁移代码逻辑。

## 2. The Kitchen Loop: User-Spec-Driven Development for a Self-Evolving Codebase

**Topics:** Agent Skill, Memory, RL, Evolve
**Authors:** Yannick Roy
**Venue / Year:** 2026
**Links:** [Abstract](http://arxiv.org/abs/2603.25697v1) | [PDF](http://arxiv.org/pdf/2603.25697v1.pdf) | [Code](https://github.com/0xagentkitchen/kitchenloop)

### 1. 一句话结论

The Kitchen Loop框架通过用户规范驱动、合成用户测试和不可伪造的验证，实现自主演化的软件系统，确保长期安全和质量提升。

### 2. 为什么重要

代码生成已非瓶颈，关键在于知道该构建什么以及证明其有效。该工作提出了一种生产级的自主演化框架，通过统一的信任模型（规范表面、合成用户测试、不可伪造测试和漂移控制）解决了AI辅助开发中的质量退化和验证不足问题，实现了零回归和持续改进。

### 3. 研究问题

1. 如何在不依赖人工审查的情况下，安全地实现软件的长期自主演化？
2. 如何验证AI生成的代码真正满足用户需求并保持质量？
3. 如何构建一个能够自我纠正和自我改进的系统？

### 4. 背景与问题设定

随着LLM编码代理的出现，代码生产变得容易，但瓶颈转移到了确定要构建什么以及证明其正确性上。现有的AI辅助开发方法存在质量退化、验证不足和速度-质量悖论等问题。传统的任务完成模式（如修复单个问题）可能导致局部修复破坏全局行为，而基于代理指标的优化则可能导致对代理指标的优化而产品在未测量的维度上退化。

### 5. 方法概览

- 任务 / 环境: 一个六阶段的自主改进循环：待办事项梳理、构思、使用、执行、润色、评审与合并。系统在生产环境中运行，通过LLM代理作为合成的超级用户，以1000倍于人类的速度在产品的规范表面上进行操作。
- 变量 / 干预设计: 通过三个核心组件构建统一的信任模型：(1) 规范表面（产品声称支持的功能集合）；(2) “作为用户×1000”（LLM代理以合成超级用户的身份进行操作）；(3) 不可伪造测试（基于真实世界状态的对抗性验证，作者无法伪造）；(4) 漂移控制（持续的质量测量和自动暂停门）。
- 评价指标: 合并的拉取请求数量、回归数量（由回归检测器检测）、质量门（L1-L4测试通过率）、成本/PR、迭代速度、每月成本、生产事故数量。
- 模型比较: 与任务完成模式（如SWE-agent）不同，Kitchen Loop采用覆盖耗尽模式，系统性地遍历规范表面的每个组合，直到覆盖差距接近零。与自我改进代理（如SICA）不同，其自我改进是锚定在规范表面和回归检测器上，而非优化目标。
- 我的理解: 该框架的核心创新在于将多个已知的AI工程实践（如合成用户、对抗性测试、漂移控制）整合成一个生产级的、具有操作纪律的系统。它通过强制性的、不可绕过的验证（不可伪造测试）和持续的质量监控（漂移控制），解决了自主演化中的信任和安全问题。其“自扩展”属性（通过三层次模型）是实现长期演化的关键。

### 6. 关键发现

- 零回归的生产验证: 在两个生产系统上进行了285+次迭代，产生了1,094+个合并的拉取请求，回归检测器未检测到任何回归，证明了该框架在生产环境中的有效性。
- 涌现的自修正链: 在大规模运行中观察到多迭代自修正链，即系统能够发现并修复自身基础设施的错误，实现了自主的基础设施修复。
- 质量门的持续提升: 质量门（L1-L4测试通过率）从76-91%单调提升至100%，表明系统质量在持续改进。

## 3. Social Hippocampus Memory Learning

### 1. 一句话结论

SoHip提出了一种基于记忆共享的社交机器学习框架，通过海马体启发的记忆整合与个体-集体记忆融合，实现异构代理间的隐私保护协作学习，最高提升8.78%准确率。

### 2. 为什么重要

现有异构联邦学习方法依赖共享模型参数或中间表示，存在隐私泄露和高开销风险。SoHip通过仅交换轻量级记忆，实现了数据与模型隐私保护下的高效协作，为异构环境下的社会学习提供了新范式。

### 3. 研究问题

1. 如何在不共享原始数据或模型参数的情况下，实现异构学习代理间的有效协作？

### 4. 背景与问题设定

社会学习强调个体通过互动和知识交换改进。在机器学习中，这催生了社交机器学习（SML），其中多个代理通过共享抽象知识协作学习。联邦学习（FL）为SML提供了基础，但现有异构FL方法常依赖共享模型参数或中间表示，可能暴露敏感信息并增加开销。

**Topics:** Agent Skill, Memory
**Authors:** Abhishek Bhandwaldar, Mihir Choudhury, Ruchir Puri, Akash Srivastava
**Venue / Year:**

### 1. 一句话结论

通过代理工厂框架，通用编码代理在无需硬件特定训练的情况下，能显著优化硬件设计，实现平均8.27倍的加速，尤其在复杂基准上效果显著。

### 2. 为什么重要

该研究展示了通用编码代理在硬件优化中的潜力，证明了代理规模扩展可以有效探索设计空间，为自动化硬件设计提供了新范式。

### 3. 研究问题

### 4. 背景与问题设定

高阶综合（HLS）旨在将硬件设计抽象化，但当前工具仍需大量专家驱动的指令插入和代码重构才能实现理想结果。现有自动化方法通常将优化视为预定义参数空间内的黑箱问题，无法进行开放式的程序变换。本研究探讨通用编码代理（无硬件特定训练）在从高级算法规范优化硬件设计方面的潜力。

### 5. 方法概览

- 任务 / 环境: 给定高级语言（如C/C++）编写的输入设计和目标实现（如FPGA平台），目标是找到最小化执行延迟并满足面积约束的实现。
- 变量 / 干预设计: 两阶段管道：第一阶段将设计分解为子内核，独立优化每个子内核并通过整数线性规划（ILP）组装全局有希望的配置；第二阶段在ILP的前N个解上启动N个专家代理，探索跨功能优化。
- 评价指标: 速度提升（相对于基线）和面积（资源利用率）
- 模型比较: 使用Claude Code (Opus 4.5/4.6)与AMD Vitis HLS进行评估，与基于受限指令集的基线进行比较。
- 我的理解: 该方法通过两阶段策略，先在子内核层面进行独立优化并用ILP进行全局筛选，再通过多个代理在顶层解上进行全局探索，有效利用了代理的规模扩展来提升优化质量。

### 6. 关键发现

- 代理规模提升探索效果: 从1到10个代理的扩展，平均速度提升8.27倍，复杂工作负载（如streamcluster）的提升超过20倍。

## 5. PICon: A Multi-Turn Interrogation Framework for Evaluating Persona Agent Consistency

**Topics:** Agent, Skill, Memory, Evaluation
**Authors:** Minseo Kim, Sujeong Im, Junseong Choi, Junhee Lee, Chaeeun Shim, Edward Choi
**Venue / Year:** 
**Links:** [Abstract](http://arxiv.org/abs/2603.25620v1) | [PDF](http://arxiv.org/pdf/2603.25620v1.pdf) | [Code](https://kaist-edlab.github.io/picon/)

### 1. 一句话结论

提出PICon框架，通过逻辑链式多轮追问评估persona代理在内部、外部和重测一致性三个维度的表现，发现现有代理在所有维度上均未达到人类基线水平。

### 2. 为什么重要

随着基于大语言模型的persona代理被广泛用作人类参与者的可扩展代理，缺乏系统方法验证其响应在交互过程中是否保持无矛盾和事实准确性。PICon提供了概念基础和实用方法，确保在信任代理作为人类替代品之前进行评估。

### 3. 研究问题

1. 如何系统地验证persona代理的响应在交互过程中是否保持无矛盾和事实准确性？
2. 现有评估方法在哪些方面存在局限性？
3. PICon框架如何通过多轮追问评估一致性？
4. 现有persona代理在内部、外部和重测一致性上表现如何？

### 4. 背景与问题设定

大型语言模型（LLM）驱动的persona代理正被迅速采用作为跨多个领域的可扩展人类参与者代理。然而，目前尚无系统方法来验证persona代理的响应在交互过程中是否保持无矛盾和事实错误。基于审讯方法的原则，即无论虚构身份多么复杂，系统性审讯都会暴露其矛盾，本文提出PICon，一个通过逻辑链式多轮提问来探测persona代理的评估框架。

### 5. 方法概览

- 任务 / 环境: PICon框架操作于两个阶段：审讯阶段和评估阶段。审讯阶段包括三个阶段：(1) 了解，提出预定义的基数人口问题；(2) 主要审讯，问题者生成后续问题，实体和声明提取器识别可验证的实体和声明，问题者通过网络搜索检索证据以生成确认问题；(3) 重测，重新提出早期问题。评估阶段评估收集的响应在内部一致性、外部一致性和重测一致性方面的表现。
- 变量 / 干预设计: 通过逻辑链式多轮提问，系统性地询问代理关于其自身的信息，并收集真实世界证据来验证其声明。
- 评价指标: 内部一致性（IC）：通过合作性和非矛盾率的调和平均值来衡量；外部一致性（EC）：通过非反驳率和覆盖率的调和平均值来衡量；重测一致性（RC）：通过原始响应和重新提出响应的匹配度来衡量。
- 模型比较: 评估了七个不同方法生成的persona代理组，包括Character.ai、OpenCharacter、Consistent LLM、Twin 2K 500、DeepPersona、Li et al. (2025)和Human Simulacra，并与63名真实人类参与者进行比较。
- 我的理解: PICon框架通过模拟审讯过程，系统性地评估persona代理在多个维度上的一致性，揭示了现有代理在面对复杂追问时的局限性。

### 6. 关键发现

- PICon框架的提出: 提出PICon，一个基于审讯方法的评估框架，通过逻辑连接的多轮提问评估persona代理的一致性，涵盖内部、外部和重测一致性三个维度。
- 系统比较: 首次系统比较了不同方法生成的persona代理在一致性上的表现，评估了七个组的persona代理和63名真实人类参与者。
- 失败模式识别: 识别了所有三个一致性维度上的不同失败模式，发现没有当前的persona代理在所有维度上都表现出色。
